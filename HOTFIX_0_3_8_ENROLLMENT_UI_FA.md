# Hotfix 0.3.8 - Enrollment UI and Server Error Cleanup

## تغییرات ویندوز

- هنگام دریافت دستور `ENROLL_FINGER` از سرور، یک پنجره واضح نمایش داده می‌شود:
  - نام فرد / شناسه کارمند
  - کد انگشت
  - کد دستگاه
  - پیام «انگشت را روی سنسور قرار دهید»
- نتیجه ثبت اثر انگشت در همان پنجره و کارت «آخرین دستور سرور» نمایش داده می‌شود.
- `pull-commands` اگر خطای 502/503/504 بدهد دیگر کل UI را با HTML خام خراب نمی‌کند.
- خطای HTML خام مثل `<!DOCTYPE html>` دیگر در پایین UI نمایش داده نمی‌شود.
- اگر فقط endpoint دستورها خطا دهد، برنامه ارسال رکوردهای حضور را تا حد ممکن ادامه می‌دهد.

## قرارداد payload پیشنهادی برای دستور ثبت اثر انگشت

```json
{
  "command_type": "ENROLL_FINGER",
  "payload": {
    "finger_id": 2,
    "employee_id": 1,
    "employee_name": "نام کارمند"
  }
}
```
